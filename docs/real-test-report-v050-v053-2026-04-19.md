# v0.5.0 vs v0.5.3 真实效果实测报告（2026-04-19）

## 测试目标
- 验证“每个页面、每个按钮、每个输入、输出”的实际可用性。
- 对比 `v0.5.0` 与 `v0.5.3` 是否存在可复现的功能回退。

## 测试方式（真实执行）
1. 分别启动 `v0.5.0` 与 `v0.5.3`（本地服务实际运行）。
2. 使用 Playwright（`msedge`）真实打开页面并执行交互。
3. 自动化覆盖：
- 页面：`/`、`/search`、`/idea`、`/compare`、`/settings`、`/ai`、`/patent/smoke-pat-001`
- 输入：`input/textarea/select` 自动填值与回读
- 按钮：逐个点击并记录成功/失败
- API 输入输出：20 条核心接口链路逐条调用并校验 HTTP 结果

### 关键观察
1. 在本次源码级真实实测中，`v0.5.3` 未出现相对 `v0.5.0` 的明显整体回退。
2. 两版本共同存在的问题模式一致：
- `/compare` 页一组按钮在通用“逐按钮点击”场景下出现超时（主要集中在上传/粘贴/对比动作）
- `/idea` 页部分按钮在当前状态下不可见或不可用（`not_visible_or_disabled`）
3. 关键接口链路两版本均通过（20/20）：
- 设置保存链路
- 创意提交/查询/聊天/总结/删除链路
- 搜索链路
- AI 对比/矩阵/创造性/审查意见链路
