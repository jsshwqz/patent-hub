# Patent Hub 快速上手

这份文档给第一次接触 `patent-hub` 仓库的人准备。

目标很简单：
- 先把应用跑起来
- 知道去哪里配置
- 完成第一次搜索或分析

如果你只是想先体验功能，不需要先读完整个仓库，也不需要先搞懂 `mobile/`、`skills/`、`tests/` 这些目录。

---

## 1. 这是个什么项目

Patent Hub 可以理解成一个“专利检索 + AI 分析 + 创意验证”的工作台。

你第一次使用时，最常接触的是这几块：
- `src/`：Rust 后端主程序
- `templates/`：页面模板
- `static/`：页面样式和脚本
- `docs/`：文档

下面这些不是你第一次使用必须关心的：
- `mobile/`：移动端 Rust UI
- `skills/`：技能相关内容
- `tests/`：测试代码

---

## 2. 最推荐的启动方式

### 方式 A：直接从源码运行

适合：
- 想先在本机跑起来
- 会一点命令行
- 不急着研究打包

准备：
- 安装 [Rust](https://rustup.rs/)

运行：

```bash
git clone https://github.com/jsshwqz/patent-hub.git
cd patent-hub
cp .env.example .env
cargo run --release --bin patent-hub
```

启动后，打开：

```text
http://127.0.0.1:3000/search
```

### 方式 B：Windows 下用仓库里的启动脚本

如果你在 Windows 上，也可以直接运行：

```text
start-patent-hub.bat
```

它会自动构建并启动主程序。

---

## 3. 第一次启动后先做什么

第一次打开后，建议按这个顺序操作：

1. 先打开设置页  
   地址：`http://127.0.0.1:3000/settings`

2. 先配 AI，再考虑在线搜索

3. 回到搜索页，完成第一次检索

最省心的思路是：
- 先把 AI 配好
- 在线搜索的 Key 可以后配

---

## 4. 怎么配置最省事

### 最低可用配置

如果你只是想先体验页面和本地能力：
- 可以先不填任何 Key

这样做的结果是：
- 应用能启动
- 本地数据库相关功能能用
- 但 AI 分析和部分在线搜索能力会受限

### 推荐配置

推荐至少配置一组 AI：

```env
AI_BASE_URL=https://open.bigmodel.cn/api/paas/v4
AI_API_KEY=你的密钥
AI_MODEL=glm-4-flash
```

这组配置也可以直接在设置页面里填，不一定非要手改 `.env`。

### 在线搜索配置

在线搜索不是第一次体验的必需项，但如果你要更完整的效果，建议再补一个：
- `SERPAPI_KEY`
- 或设置页里的 Bing / Lens 配置

---

## 5. 第一次建议怎么用

### 路线 1：先做一次专利搜索

1. 打开 `/search`
2. 输入关键词、专利号、申请人或发明人
3. 先试一次本地搜索
4. 如果结果不够，再试在线搜索

### 路线 2：看一条专利详情

进入详情页后，重点看这些能力：
- 权利要求
- AI 摘要
- 相似专利
- PDF 导出
- 标签和收藏夹

### 路线 3：做一次创意验证

打开 `/idea` 后，你会看到两种模式：
- 快速验证：适合先看一个结论
- 深度验证：适合走完整的分析流程

如果你第一次体验，我建议先用“快速验证”。

### 路线 4：做一次对比

如果你已经有两条或多条专利，或者有自己的技术文档，可以去 `/compare` 或专利详情页里的上传对比功能。

## 6. 最容易卡住的地方

### 为什么搜索不到结果？

常见原因有两个：
- 你在用本地搜索，但本地数据库里还没有数据
- 你以为自己在用在线搜索，但在线搜索配置还没完成

### 为什么 AI 没有返回？

最常见原因：
- `AI_API_KEY` 没配
- `AI_BASE_URL` 或 `AI_MODEL` 配错

先去设置页检查最直接。

### 为什么一打开仓库感觉很复杂？

因为仓库里不仅有主程序，还有移动端、测试、技能和文档。

但第一次使用时，你其实只需要关心这 3 件事：
- 怎么启动
- 怎么配置
- 从哪个页面开始用

### 一定要看懂仓库结构才能用吗？

不用。

第一次使用只要记住：
- 主入口是 `cargo run --release --bin patent-hub`
- 页面入口是 `/search`
- 配置入口是 `/settings`

---

## 7. 推荐的新手路径

如果你只给自己 10 分钟，我建议这样走：

1. 启动应用
2. 打开设置页，配置 AI
3. 回到搜索页，搜一个关键词
4. 打开一条专利详情
5. 点一次 AI 分析或权利要求分析
6. 再去 `/idea` 提交一次创意验证

走完这一遍，你基本就知道这个产品是怎么用的了。
