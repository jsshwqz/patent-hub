# 发布前人工实测清单（强制）

> 目标：避免“可编译但不可用”的伪通过。
> 规则：每次发布前必须全量执行，且输出本次实测记录（通过/失败/截图/结论）。

## A. 页面可达与基础交互
1. 首页 `/` 可打开，关键按钮可点击。
2. 搜索页 `/search` 可打开，检索输入框与结果区正常。
3. 创意页 `/idea` 可打开，提交与历史区正常。
4. 对比页 `/compare` 可打开，上传/比较入口正常。
5. 设置页 `/settings` 可打开，配置可保存并回显。
6. AI 页 `/ai` 可打开，基础对话流程可触发。

## B. 核心功能输入输出（必须记录样例）
1. 创意提交：
- 输入：标题 + 描述
- 预期输出：返回 `id`，`/api/idea/list` 可见该条目

2. 创意报告：
- 输入：有效 `idea_id`
- 预期输出：`/api/idea/:id/report` 返回结构化内容，无 500

3. 创意聊天：
- 输入：有效消息
- 预期输出：`/api/idea/:id/chat` 返回响应；`/messages` 可取回记录

4. 创意删除：
- 输入：有效 `idea_id`
- 预期输出：删除成功，列表中不可见

5. 检索功能：
- 输入：关键词与专利号各 1 组
- 预期输出：结果结构完整、无前端崩溃、无后端 500

6. 对比功能：
- 输入：至少 2 份样本（文本/PDF）
- 预期输出：可生成对比结果，关键字段有值

## C. 非功能验收
1. `cargo fmt --all` 通过。
2. `cargo test --tests` 通过。
3. `cargo clippy --tests -- -D warnings` 通过。
4. 启动后日志无连续报错（同类错误连续 3 次视为失败）。
5. 深度门禁必须通过（见 `docs/depth-quality-gate.md`）：
- 执行：`python tools/depth_gate_test.py --worktree D:\test\innoforge-v053 --bin innoforge --out docs/depth-gate-runs/run-<timestamp>`
- 要求：4 个 AI 深度用例全部通过，且 `gate_pass=true`
- 失败即不可发布，不允许以“HTTP 200/有回复”替代通过

## D. 发布链路验收（双端）
1. GitHub 与 Gitee `main` HEAD 一致。
2. GitHub Release 资产完整（平台包数量与命名符合约定）。
3. Tag 指向与发布说明一致。
4. 发布后复验上述 A/B 关键路径至少一轮。
